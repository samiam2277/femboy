## 1. 产品概述

### 1.1 产品定位

一款**文字人生模拟器**。玩家扮演一名生理性别为男、对女性化事物有天然亲近感的角色，通过开局天赋选择、人生事件抉择、属性浮动，体验从童年到中年的关键人生节点，最终迎来不同结局。

**核心体验**：在随机与选择中，看见一个"男娘"可能经历的各种人生——有的温暖，有的残酷，有的荒诞，有的平淡。

### 1.2 一句话卖点

> 重开一百次，体验一百种不被允许的人生。

### 1.3 竞品分析

| 竞品 | 类型 | 我们的差异化 |
|------|------|-------------|
| 《人生重开模拟器》 | 文字人生模拟 | 聚焦亚文化身份议题，事件更写实、更扎心 |
| 《修仙模拟器》 | 文字修仙模拟 | 修仙是修为突破，我们是身份认同与社会压力的博弈 |
| 《中国式家长》 | 回合制养成 | 我们无复杂操作，纯事件驱动，单局更快 |
| 《Beholder》 | 叙事选择 | 我们选择更私人化、内向化，非政治惊悚 |

## 2. 目标用户

### 2.1 核心用户

- **文字游戏/模拟器爱好者**：玩过《人生重开模拟器》《修仙模拟器》《瘟疫公司》等
- **二次元亚文化参与者**：对男娘/cos/女装文化有认知或兴趣
- **年龄**：16-28 岁
- **平台**：PC（Steam）+ 移动端（微信小程序/网页/H5）+ 网页版

### 2.2 扩展用户

- 对"非主流人生路径"有好奇心的泛玩家
- 轻量级玩家（每天玩 1-2 局，每局 10 分钟）

### 2.3 非目标用户

- 寻求重度策略或操作感的玩家
- 寻求成人内容或猎奇体验的玩家

## 3. 核心架构

### 3.1 核心玩法循环

```
开局：天赋抽取（3选/10点制） → 初始属性生成 →
人生推进：每年触发 1 个事件 → 事件描述 + 2-4 个选项 →
选择 → 属性变动 + 可能触发连锁事件 →
关键节点（升学、恋爱、出柜等）：多选项高代价 →
死亡/结局触发 → 人生总结 + 结局称号 + 天赋/成就解锁 →
重开（可继承已解锁天赋）
```

### 3.2 单局流程

| 阶段 | 内容 | 时长 |
|------|------|------|
| **天赋选择** | 抽取/选择 3 个天赋 | 1-2 分钟 |
| **属性分配** | 10 点自由分配到 6 维属性 | 1 分钟 |
| **人生推进** | 约 20-40 个事件，覆盖 6-45 岁 | 6-12 分钟 |
| **结局结算** | 人生总结、结局称号、数据统计 | 1-2 分钟 |
| **解锁反馈** | 新天赋/成就/事件图鉴 | 1 分钟 |

### 3.3 重开价值

- **天赋收集**：数百个天赋，部分需特定结局解锁
- **事件图鉴**：收集全部事件的触发条件
- **成就系统**：各种极端结局的达成
- **排行榜（可选）**：单局评分排名

## 4. 详细系统设计

### 4.1 天赋系统

#### 4.1.1 天赋品质

| 品质 | 颜色 | 出现概率 | 说明 |
|------|------|----------|------|
| 普通 | 白 | 60% | 微调单项属性 |
| 稀有 | 蓝 | 25% | 显著属性加成或解锁事件池 |
| 史诗 | 紫 | 12% | 改变核心机制或开启隐藏线 |
| 传说 | 金 | 3% | 极端开局，高风险高回报 |

#### 4.1.2 天赋示例

| 天赋名 | 品质 | 效果 |
|--------|------|------|
| 天赋党 | 蓝 | 颜值 +15，骨架底子视为极佳 |
| 开明家庭 | 紫 | 家庭线事件全部正向，心理压力 -30% |
| 二次元世家 | 蓝 | 才华 +5，童年自动接触 cosplay |
| 药娘之路 | 金 | 解锁激素事件链，颜值成长极快，但健康逐年衰减 |
| 钢铁肠胃 | 白 | 健康 +5 |
| 赛博难民 | 紫 | 网络相关事件收益翻倍，线下事件风险翻倍 |
| 照骗大师 | 蓝 | 颜值判定 +20%（仅限线上事件） |
| 毕业焦虑 | 白 | 25 岁后每年概率触发"是否退圈"抉择 |
| 富哥开局 | 蓝 | 初始财富 +30 |
| 孤勇者 | 紫 | 无法获得任何 NPC 帮助，但自洽成长翻倍 |

### 4.2 属性系统（六维）

所有属性范围 0-100，初始由天赋 + 自由分配决定。

| 属性 | 说明 | 主要影响 |
|------|------|----------|
| **颜值** | 女性化外观综合水平 | 社交反馈、恋爱事件、网络事件成功率 |
| **才华** | 学习能力、才艺、职业技能 | 工作收入、创作事件、逃脱困境选项 |
| **财富** | 有钱程度 | 消费选择、抗风险能力、部分事件的解锁 |
| **自洽** | 自我接纳与心理稳定 | 抗压、不被外界评价击垮、触发内心平静结局 |
| **社会** | 在主流社会中的融入度 | 职场顺利度、家庭关系、被接纳程度 |
| **女装度** | 女性化表达的深度与频率 | **核心特色属性**。 unlock 女装相关事件，过高触发异样眼光事件 |

#### 4.2.1 属性联动示例

- **女装度高 + 颜值低**：触发"翻车"类事件（被嘲笑、被挂）
- **女装度高 + 自洽低**：触发内心挣扎事件（罪恶感、焦虑）
- **财富低 + 社会低**：触发生存危机事件（没钱脱毛、被房东发现）
- **才华高 + 女装度高**：触发创作出路事件（成为妆娘、写经验帖）

### 4.3 事件系统（核心）

#### 4.3.1 事件结构

```
[年份] 你 XX 岁了。

[事件描述文本，100-300 字]

选项 A：[描述] → 属性变动 / 触发新事件 / 死亡
选项 B：[描述] → ...
选项 C（条件解锁）：[描述，需特定属性或前置事件] → ...
```

#### 4.3.2 事件池分类

| 类别 | 数量（估算） | 触发方式 |
|------|-------------|----------|
| **童年事件池** | 15-20 个 | 随机抽取，6-12 岁 |
| **青春期事件池** | 25-30 个 | 随机 + 属性条件，13-18 岁 |
| **成年事件池** | 25-30 个 | 随机 + 属性条件，19-35 岁 |
| **中年事件池** | 10-15 个 | 随机 + 属性条件，36-50 岁 |
| **关键事件** | 15-20 个 | 满足条件必触发，跨年龄段 |
| **连锁事件** | 10-15 个 | 前置事件触发后，数年后回访 |
| **死亡事件** | 10 个 | 属性极端或随机触发 |

**总计**：约 110-140 个独立事件。

#### 4.3.3 关键事件示例

| 事件名 | 触发条件 | 核心抉择 |
|--------|----------|----------|
| **第一次** | 女装度 > 10，年龄 ≥ 10 | 偷偷穿 / 忍不住穿了又脱 / 被发现 |
| **网吧耳机** | 青春期，在网吧看女装视频被邻座发现 | 否认 / 沉默 / 反问 |
| **漫展初体验** | 女装度 > 20，年龄 ≥ 15 | 去 / 不去 / 去了但只围观 |
| **坦白之夜** | 社会 > 40 或特定 NPC 关系达标 | 向家人 / 朋友 / 恋人坦白 |
| **厕所战争** | 女装出门，年龄 ≥ 16 | 去男厕 / 去女厕 / 忍着回家 |
| **金主出现** | 财富 < 20，女装度 > 40，网络事件后 | 接受资助 / 拒绝 / 报警 |
| **退圈抉择** | 年龄 ≥ 25，每年概率触发 | 彻底退圈 / 减少频率 / 坚持 |
| **母亲的搜索记录** | 家庭线未坦白，年龄 ≥ 18 | 发现母亲搜索了"儿子喜欢穿女装怎么办" |

#### 4.3.4 连锁事件示例

```
初中时你向同桌小 X 坦白 → 埋下伏笔
    ↓ 5 年后
小 X 成为了你的公司 HR / 你的恋人闺蜜 / 某个挂人帖的发起者
```

#### 4.3.5 文案风格

- **扎心向**："你花了三小时化妆，在镜子前觉得自己今天真的很好看。出门十分钟后，地铁上一个小孩大声问他妈妈：那个姐姐为什么是男的？"
- **温暖向**："你把链接发给网友 A，说'这是我第一次发没 P 的'。对方回：'早就看出来了，但没关系，你笑起来很好看。'"
- **荒诞向**："你在二手平台卖闲置女装，买家约线下交易。见面发现是高中班主任。"
- **平淡向**："又一个周末。你化好妆，拍了几张照片，修了半小时图，发了微博，收获 12 个赞。然后卸妆，洗脸，回归日常。"

### 4.4 结局系统

#### 4.4.1 结局触发

- **自然死亡**：年龄 60-80 随机（受健康/自洽/财富影响）
- **意外死亡**：属性极端（如自洽=0 触发自杀，财富=0 + 女装度高触发恶性事件）
- **特殊结算**：达成特定条件提前进入结局（如彻底退圈、手术完成、成为公众人物）

#### 4.4.2 结局结算内容

1. **人生总结**：3-5 句话概括这一生
2. **结局称号**：如"平凡而幸福的小满人生""赛博偶像的黄昏""无人知晓的玫瑰""行尸走肉的四十年"
3. **属性雷达图**：六维最终值可视化
4. **关键选择回顾**：这一生最重要的 3 个选择
5. **解锁内容**：新天赋 / 新事件 / 成就徽章

#### 4.4.3 结局示例（矩阵分布）

| | **高自洽** | **低自洽** |
|---|---|---|
| **高女装度** | A-星辰：自洽的女装偶像/艺术家 | B-燃尽：燃烧后熄灭的流星 |
| **中女装度** | C-小满：有同温层、有理解的普通人 | D-困兽：在分裂中磨损的人 |
| **低女装度** | E-和解：放下后找到其他热爱的退圈者 | F-行尸：彻底阉割自我后的空心人 |

**特殊结局**：
- **药娘的终点**：选择激素线后的多种分支（成功过渡 / 健康崩溃 / 卡在中间态）
- **赛博飞升**：网络人格彻底取代现实身份
- **未发送的短信**：终章前夜的撤回，留下开放性
- **轮回**：下一代角色继承了你的某个天赋

### 4.5 蝴蝶效应（简化版）

v2 不采用 v1 的复杂伏笔引擎，而是：

- **关键选择标记**：在特定事件中做出选择后，系统记录标记
- **数年后回访**：在后续年龄段事件中，根据标记改变文本或选项
- **简单连锁**：A 事件 → B 事件（1-3 年后）→ C 结局条件

示例：
- 初中选择"向同桌坦白" → 高中事件"同桌的朋友圈" → 成年事件"意外重逢"

## 5. 叙事与内容结构

### 5.1 人生阶段

| 阶段 | 年龄 | 事件密度 | 基调 |
|------|------|----------|------|
| **童年** | 6-12 | 稀疏（3-5 个事件） | 懵懂、好奇、初识规训 |
| **青春期** | 13-18 | 密集（8-12 个事件） | 焦虑、觉醒、初恋、群体归属 |
| **成年初显** | 19-25 | 密集（8-10 个事件） | 独立、抉择、网络、恋爱 |
| **建立期** | 26-40 | 中等（6-8 个事件） | 沉降、转型、家庭、代际 |
| **中年+** | 41-终 | 稀疏（2-4 个事件） | 回望、和解、 legacy |

### 5.2 事件编写原则

1. **真实优先**：每个事件都可以在现实社群中找到原型
2. **无评判**：不暗示哪个选择是"正确"的，只呈现后果
3. **圈层用语**：自然使用"集邮""返图""翻车""毕业""药娘""chaser"等术语
4. **具体而非诗意**："假睫毛胶是不是要开了" 优于 "我如蝴蝶般破碎"
5. **多样性**：同一事件有多种触发方式和多种结果，避免重复感

### 5.3 角色/NPC 事件化

v2 不设独立攻略系统，NPC 作为**事件中的角色**出现：

| NPC | 出现场景 | 功能 |
|-----|----------|------|
| **母亲** | 家庭线贯穿事件 | 从不理解到沉默接纳，或终生不提 |
| **同桌/同学** | 校园事件 | 可能是第一个发现者，也可能是第一个嘲笑者 |
| **网友 A** | 网络事件 | 知己或骗子 |
| **前辈（Lisa 姐型）** | 成年事件 | 提供"退圈"或"坚持"的参照 |
| **恋人** | 恋爱事件 | 接受 / 有条件接受 / 无法接受 |
| **摄影师/金主** | 灰色事件 | 行业内潜规则的具象化 |

## 6. 美术与表现规格

### 6.1 视觉风格

- **核心**：文字为主，一切服务于阅读体验
- **参考**：《人生重开模拟器》的简洁 + 《Replica》的信息界面感
- **色调**：深色模式为主（#1a1a2e 背景 + #eee 文字），减少阅读疲劳

### 6.2 界面构成

| 元素 | 需求 | 优先级 |
|------|------|--------|
| **主背景** | 纯色/极简渐变，可能带微弱纹理 | 必须 |
| **事件文本区** | 居中或偏左大段文字，支持打字机效果 | 必须 |
| **选项按钮** | 2-4 个，简洁矩形 | 必须 |
| **属性面板** | 六维进度条，局内常驻或呼出 | 必须 |
| **年份/年龄指示** | 顶部显示 "你 XX 岁了" | 必须 |
| **角色剪影** | 事件相关角色的极简剪影或 icon | 高 |
| **背景氛围图** | 每阶段 1-2 张氛围图（教室/漫展/出租屋/办公室） | 中 |
| **结局插画** | 每个主要结局 1 张静态图 | 中 |
| **天赋图标** | 白蓝紫金品质各一套 icon | 低 |

### 6.3 动画与动效

- **打字机效果**：事件文本逐字出现（可跳过）
- **属性跳动**：选择后属性数值 ± 浮动动画
- **年份切换**：简单淡入淡出
- **死亡/结局**：文字变灰/变红，背景音乐渐弱

### 6.4 美术量估算

| 资源 | 数量 | 风格 |
|------|------|------|
| 氛围背景图 | 8-12 张 | 极简插画/摄影滤镜 |
| 结局插画 | 15-20 张 | 情绪化静态插画 |
| NPC 剪影/头像 | 8-10 个 | 极简 icon 风 |
| UI 素材 | 1 套 | 扁平/新拟态 |

**总计**：远低于 v1 的 300-500 张立绘差分。

- DLC 同步移动端剧情包

## 9. 技术需求

### 9.1 推荐技术栈

| 平台 | 方案 | 理由 |
|------|------|------|
| **H5/微信小程序** | Vue/React + 纯前端存档 | 最快上线，便于传播 |
| **PC（Steam）** | Electron 封装 Web 版 | 代码复用，低成本移植 |
| **移动端 App** | React Native / Flutter | 若需原生体验 |

**无需游戏引擎**：纯事件驱动的文字游戏，用前端框架 + JSON 配置表即可。

### 9.2 核心技术架构

```
事件系统：JSON 配置表驱动
    - 事件 ID、触发条件（属性/年龄/前置事件）、文本、选项、结果
属性系统：六维数值 + 简单加减
存档系统：localStorage / 云存档（微信/Steam）
多语言：i18n JSON，核心为中英日韩
```

### 9.3 关键工具

- **事件编辑器**：Excel / Google Sheets → 导出 JSON，供策划直接填写
- **AB 测试框架**：事件选项的热度分析（可选）

## 10. 风险评估与红线

### 10.1 内容红线

| 红线 | 处理方式 |
|------|----------|
| 未成年人色情 | 所有恋爱/身体相关事件设定在 18+ |

### 10.2 运营风险

| 风险 | 应对 |
|------|------|
| 微信平台审查 | 避免敏感词直出，用圈层黑话/隐喻；备好备用文案包 |
| 圈外差评 | 标签精准化，不主动推送至非目标用户 |
| 圈内争议 | 不站队，不评判，只呈现选择后果 |

- **ESRB**：T（13+）或 M（17+）
- **国内**：16+
